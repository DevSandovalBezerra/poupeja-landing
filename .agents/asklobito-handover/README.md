# AskLobito RAG Handover

Este pacote existe para portar para o `asklobito` os mesmos avancos de RAG ja consolidados no `asktel`, sem depender de memoria oral, contexto perdido ou "acho que era assim".

## Ordem de leitura

1. `01-handover-executivo.md`
2. `02-rag-parity-blueprint.md`
3. `03-mapa-de-arquivos-asktel.md`
4. `04-checklist-de-implantacao.md`
5. `05-validacao-e-go-live.md`

## Arquivos de apoio

- `task_plan.md`: plano de execucao resumido
- `findings.md`: descobertas criticas do estado atual do `asktel`
- `progress.md`: log do pacote entregue neste handover

## Resultado esperado

Ao terminar esta leitura, qualquer pessoa do time deve conseguir:

- entender o que o RAG do `asktel` faz hoje de verdade;
- saber quais arquivos e tabelas sustentam isso;
- reimplementar no `asklobito` sem pular dependencias;
- validar a paridade com testes e checks operacionais.
